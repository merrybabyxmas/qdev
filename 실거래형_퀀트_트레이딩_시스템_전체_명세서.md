# 실거래형 퀀트 트레이딩 시스템 전체 명세서

## 0. 문서 목적
이 문서는 다음 목표를 만족하는 **연구-개발-검증-배포 일체형 퀀트 트레이딩 시스템**의 상위 명세서이다.

- 금융지식, 금융시장, 포트폴리오, 트레이딩, 수학이론, 딥러닝을 통합한다.
- 관련 서적, 논문, 알고리즘 후보군을 체계적으로 정리한다.
- VSCode 서버 환경에서 재현 가능한 연구/개발/실행 환경을 구성한다.
- 백테스트, 페이퍼트레이딩, 실거래를 단계적으로 연결한다.
- 수익률만이 아니라 **낙폭, 변동성, 거래비용, 슬리피지, 체결 리스크, 레짐 변화**까지 함께 고려한다.
- 실제 계좌 연동이 가능하도록 브로커 추상화 계층을 둔다.
- 모든 실험, 전략 후보, 데이터 후보, 모델 후보, 평가 결과를 문서화하고 버전 관리한다.

본 시스템의 절대 원칙은 다음과 같다.

1. **실거래 이전에 반드시 페이퍼트레이딩과 운영 리스크 검증을 완료한다.**
2. **단일 전략 올인은 금지하고, 전략 다변화 및 리스크 한도 관리가 우선이다.**
3. **최대 수익이 아니라 위험조정수익의 안정적 향상을 목표로 한다.**
4. **새로운 데이터가 들어와도 전체 시스템이 자동으로 무너지지 않도록 재학습/재평가 규칙을 명확히 정의한다.**

## 1. 최종 목표 정의

### 1.1 상위 목표
- 최소한의 리스크로 최대한의 수익을 추구하는 자동 트레이딩 시스템 구축
- 연구 문서와 실제 코드가 일치하는 운영 가능한 아키텍처 구축
- 시장 상태별 전략 전환 혹은 앙상블 의사결정 체계 구축
- 실시간 데이터 유입 시 예측, 포지션 조정, 주문 집행, 로그 기록, 리스크 점검 자동화

### 1.2 현실적인 재정의
“무조건 최소 리스크, 최대 수익”은 수학적으로 동시 달성이 불가능한 이상 목표에 가깝다. 따라서 실무 목표는 아래와 같이 재정의한다.

- 최대화: 연환산 수익률, 샤프 비율, 칼마 비율, 소르티노 비율
- 최소화: 최대 낙폭, tail risk, turnover, 거래비용 민감도, 체결 실패 가능성
- 제약조건: 자산/섹터/전략별 노출 한도, 일일 손실 한도, 레버리지 한도, 변동성 타깃, 상관 구조 집중도 제한

즉, 본 시스템은 **제약 조건이 있는 위험조정수익 최적화 문제**로 정의된다.

## 2. 전체 시스템 아키텍처

### 2.1 계층 구조
1. **Knowledge Layer**: 책/논문/전략 아이디어/가설 저장
2. **Data Layer**: 가격, 거래량, 재무, 뉴스, 거시경제, 대체데이터 수집 및 정제
3. **Research Layer**: 피처 생성, 모델링, 전략 생성, 가설 검증
4. **Backtest Layer**: 비용/슬리피지 반영 시뮬레이션
5. **Portfolio Layer**: 종목 선택, 비중 결정, 리밸런싱
6. **Execution Layer**: 주문 생성, 체결 관리, 브로커 연동
7. **Risk Layer**: 손실 제한, 포지션 제한, 이상 탐지, 킬스위치
8. **Monitoring Layer**: 대시보드, 알람, 로그, 리포트
9. **Deployment Layer**: 실서버 운영, 스케줄링, 재시작, 비밀키 관리

## 15. 새 데이터 유입 시 활용 방식

### 15.1 온라인 업데이트 레벨
- Level 0: 데이터만 저장, 다음 배치 학습에 사용
- Level 1: 피처만 즉시 갱신, 모델은 고정
- Level 2: confidence calibration만 갱신
- Level 3: 경량 온라인 학습
- Level 4: full retraining + revalidation 후 배포

### 15.2 선택 기준
- 데이터 드리프트 크기
- 성능 저하 정도
- 시장 레짐 급변 여부
- 전략 민감도
- 운영 시간대

### 15.3 권장 원칙
실거래에서는 기본적으로 **즉시 full retraining 금지**. 먼저 shadow model로 검증 후 승격한다.

## 19. 최종 추천 시스템 형태
가장 현실적인 첫 번째 실전 시스템은 다음 조합이 적절하다.

1. **시장 레짐 분류기**: HMM 또는 변동성/상관 기반 규칙 모델
2. **예측 엔진**: XGBoost + Transformer 계열 앙상블
3. **포트폴리오 엔진**: HRP 또는 Risk Parity 기반 안정 배분
4. **리스크 엔진**: 변동성 타깃 + drawdown cutoff + exposure cap
5. **실행 엔진**: 유동성 필터 + limit/market hybrid + TWAP fallback
6. **운영 모드**: batch retrain + paper trading 검증 후 live 승격

이는 순수 딥러닝 올인보다 더 안정적이고, 순수 규칙 기반보다 확장성이 좋으며, RL 단독보다 운영 리스크가 낮다.
